# DriftDraw

An async multiplayer "telephone" drawing party game. Players write captions, an
AI redraws each caption into an image, and the picture **drifts** hilariously as
it passes around the group. At the end, every chain is revealed start-to-finish
and players vote on the funniest result.

Built for an AI Challenge, where AI is woven directly into gameplay — not just
used as a build tool:

- **AI is the artist.** Every drawing step is generated by Google's image model
  (Imagen / Gemini), so no one has to draw.
- **AI keeps the game moving.** If a player misses their turn before the
  deadline, Claude (with vision) looks at the image and writes a caption in their
  place, so a slow player never stalls the whole game.

## Game description

DriftDraw is a Gartic-Phone-style telephone game for 2+ players (sweet spot 3–8),
played asynchronously through a shareable web link — no app install, no accounts.

1. A **host** creates a game and shares the invite link.
2. Players open the link, pick a display name, and join the lobby.
3. Each player seeds **one chain** with a short caption (e.g. "a cat doing taxes").
4. The AI renders each caption into an image; the chain is handed to the next
   player, who sees **only that image** and writes a new caption for it.
5. Every chain passes through every player exactly once. The accumulated "drift"
   between the original idea and the final image is the whole point.
6. When all chains complete, each one is **revealed** start-to-finish and players
   **vote** on the funniest.

## API

| Method + path | Body / query | Purpose |
|---|---|---|
| `POST /api/games` | `{ hostName, turnDeadlineMs }` | Create a game |
| `POST /api/games/:id/players` | `{ name }` | Join a game |
| `POST /api/games/:id/start` | — | Host starts the game |
| `GET /api/games/:id?playerId=…` | — | Player-scoped state + pending tasks |
| `POST /api/games/:id/captions` | `{ playerId, stepId, text }` | Submit a caption |

## Setup

**Prerequisites:** Node.js 20+ and npm.

```bash
git clone <this-repo>
cd ai-challenge-kenny-lee
npm install
```

To run the live API (not required for tests), copy `.env.example` to `.env` and
fill in the keys — an Anthropic API key, a Google AI Studio (Gemini) key, and a
Supabase project URL + service key. Deploying the app to the cloud is documented
separately in [`DEPLOYMENT.md`](DEPLOYMENT.md) (Vercel + Supabase).

> **Note on local installs:** this project was developed on a machine behind a
> corporate web filter that blocks `npm install` from the public npm registry.
> Dependency installs, tests, and builds therefore run in a clean-internet
> environment (GitLab CI and Vercel). If `npm install` fails on your network,
> that's the likely cause. See [`RETROSPECTIVE.md`](RETROSPECTIVE.md).

## Run

```bash
npm test         # run the full Vitest suite (108 tests) — the main correctness signal
npm run typecheck # TypeScript type-check with no emit
npm run dev      # start the Next.js dev server (needs env vars + the UI slice)
npm run build    # production build (runs on Vercel)
```

The test suite is the recommended way to see the project working today: it drives
a full create → join → start → play → reveal game through the API handlers with
mock AI, offline and free.
